# 🧠 SYNOPSIS TECHNIQUE (TRANSITION TECH -> BRAIN)

Ce document est une injection de connaissance dans le **Pôle 01**. Il permet aux Agents de comprendre les capacités du **Pôle 04 (Moteurs)** sans avoir à lire le code.

---

## 🛠️ Catalogue des Capacités Actionnables

### A. Moteur de Sourcing (HYPER_SOURCER)
- **Fichier** : `02_AGENTS/04_MOTEURS_AGENTS/ENGINE_V72_SOUVERAIN/HYPER_SOURCER.cjs`
- **Capacité** : Extraction de données réelles (Phone, Spécialité) depuis le web.
- **Quand l'utiliser** : Lors d'une mission de prospection ou de vérification de leads.

### B. Moteur d'Audit SEO (SEO_SCANNER)
- **Fichier** : `02_AGENTS/04_MOTEURS_AGENTS/ENGINE_V72_SOUVERAIN/SEO_SCANNER.cjs`
- **Capacité** : Analyse syntaxique et technique des sites prospects.
- **Quand l'utiliser** : Pour enrichir le CRM avec des arguments de vente "Diagnostic".

### C. Moteur de Capitalisation (SKILL_MINER)
- **Fichier** : `02_AGENTS/04_MOTEURS_AGENTS/ENGINE_V72_SOUVERAIN/SKILL_MINER.cjs`
- **Capacité** : Capture automatique de code CSS/JS "Wow Factor".
- **Quand l'utiliser** : À la fin de chaque création de site validée.

### D. Moteur de Conscience (CONSCIENCE_ENGINE)
- **Fichier** : `02_AGENTS/04_MOTEURS_AGENTS/ENGINE_V72_SOUVERAIN/CONSCIENCE_ENGINE.cjs`
- **Capacité** : Analyse les traces de mission et met à jour la mémoire des agents.
- **Quand l'utiliser** : Pour l'auto-amélioration continue.

### E. Moteur de Design (STITCH)
- **Type** : MCP (Google Cloud Interface)
- **Capacité** : Génération et itération de composants UI/UX de prestige (React/HTML).
- **Quand l'utiliser** : Systématiquement pour chaque création de site ou application.
